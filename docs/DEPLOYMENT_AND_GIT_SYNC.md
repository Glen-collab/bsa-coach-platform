# Deployment & Git Sync

How the BSA repos get to production, and the one rule that keeps git and the
live servers from drifting apart.

---

## The golden rule

**Git must always match what's live. Never let production code exist only on
the Desktop.**

If the only copy of running code is your local working tree, a dead drive =
unrecoverable production. So every deploy ends with git and the server in sync.

Two acceptable orders:

1. **Preferred — commit first, then deploy:** `git add` → `git commit` →
   `git push` → *then* deploy. You never deploy un-backed-up code.
2. **Fast path — deploy direct, then back up:** deploy to EC2, then immediately
   `git add` → `commit` → `push` the *same* working copy. A deploy isn't "done"
   until git matches the server.

A deploy is not finished until `git status` is clean (or the only diffs are
intentional WIP you're choosing not to ship).

---

## Two deploy models in the ecosystem

### Model A — Git-driven (Netlify auto-deploy)

**Repos:** `workoutbuilder`, `WorkoutTracker`, `react-trainer-dashboard`
(all Netlify, build on push to `main`).

Here **git push *is* the deploy.** Netlify rebuilds from the pushed commit.
Git and live can never drift — pushing is the only way to ship.

```bash
# from the repo root
git add <changed files>
git commit -m "..."
git push origin main          # Netlify auto-builds & deploys
```

`dist/` is gitignored in these repos — Netlify rebuilds it. Don't commit build
output.

### Model B — Direct-to-EC2 (manual build + scp)

**Repo:** `bsa-coach-platform` (frontend `/var/www/bestrongagain/`, backend
`/opt/bestrongagain/` on EC2 `3.19.135.182`).

Here git is **not** in the deploy path — you build locally and copy to the
server. This is the one that drifts if you forget step 0.

> **This is the model where the golden rule matters most.** Because pushing
> doesn't deploy and deploying doesn't push, you have to do both by hand.

**Step 0 (don't skip): commit + push the working copy first.**

```bash
cd ~/Desktop/bsa-coach-platform
git add -A
git commit -m "..."
git push origin master
```

**Frontend deploy** (React → `/var/www/bestrongagain/`):

```bash
npm run build
KEY=~/Desktop/polly-connect-key.pem
HOST=ec2-user@3.19.135.182
ssh -i "$KEY" "$HOST" "rm -rf /tmp/dist-new && mkdir -p /tmp/dist-new"
scp -i "$KEY" -r dist/* "$HOST:/tmp/dist-new/"
ssh -i "$KEY" "$HOST" "sudo rm -rf /var/www/bestrongagain/assets && sudo cp -r /tmp/dist-new/* /var/www/bestrongagain/"
```

**Backend deploy** (Flask `.py` or data file → `/opt/bestrongagain/`):

```bash
scp -i "$KEY" backend/<file> "$HOST:/tmp/<file>"
ssh -i "$KEY" "$HOST" "sudo mv /tmp/<file> /opt/bestrongagain/<file> && sudo systemctl restart bestrongagain.service"
```

**Verify after any EC2 deploy:**

```bash
ssh -i "$KEY" "$HOST" "sudo systemctl is-active bestrongagain.service"   # -> active
curl -s -o /dev/null -w "%{http_code}\n" https://app.bestrongagain.com/  # -> 200
```

## What Glen actually does (the honest version)

For the coach platform, Glen builds **everything from his Desktop** and deploys
**straight to EC2** via the commands above. The challenges, game-mode,
dashboards, kiosk, 2k challenge, etc. were all built and shipped this way and
are **live** — they just often sit **uncommitted** in the local repo because
the deploy never touched git.

That's exactly the drift this doc exists to prevent. Live ≠ committed. After
shipping to EC2, come back and `git push` so the repo reflects production.

## Generated artifacts that must be rebuilt, not hand-edited

- **`exercise_manifest.json`** (frontend `src/data/` + backend `backend/data/`)
  is generated by `scripts/build_exercise_manifest.js` from the
  `workoutbuilder` exercise libraries. Re-run it after any library change, then
  deploy both copies. See `reference: Exercise Video Wiring`.

  ```bash
  node scripts/build_exercise_manifest.js   # writes both copies
  ```

  Source of truth for the libraries is **`workoutbuilder/`** (the live repo),
  NOT the stale `workoutbuilder-tkd/` clone.

## Example: 2026-05-30 video-wiring deploy

1. Wired 11 new Cloudflare videos into `workoutbuilder/src/data/` →
   `git push origin main` → Netlify auto-deployed builder + tracker. *(Model A)*
2. Repointed the manifest generator at `workoutbuilder`, regenerated both
   manifest copies → committed just those 3 files →
   `git push origin master`. *(Step 0)*
3. `npm run build` → scp frontend to `/var/www/bestrongagain/`; scp backend
   manifest to `/opt/bestrongagain/data/` → restart service. *(Model B)*
4. Verified: service `active`, site `HTTP 200`, manifest `total: 1961`.
